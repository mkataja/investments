# HTTP API and client behavior

Routes, CORS, validation: **`api`** entrypoint — single source of truth; do not duplicate a route list outside code.

This doc: imports, instruments, transaction CRUD, portfolio distributions. Portfolios, brokers, transaction listing, positions, health: [Other routes](#other-routes). Edge cases and status codes: **`api/src/index.ts`**.

- `GET /instruments` — full instrument rows (`isin`, `holdings_distribution_url`, `provider_breakdown_data_url` when set), `netQuantity` (buys − sells), optional joined `distribution` (`fetchedAt`, `source`, `payload` with `countries` / `sectors`, optional `yahooFinance` / `seligsonDistribution` raw), optional `providerHoldings` (`source`, `fetchedAt`, `raw` — CSV text or base64 XLSX), optional `broker` (`id`, `name`, `brokerType`) when `broker_id` set, optional `seligsonFund` (`id`, `fid`, `name`) for Seligson-linked `custom`. Query `brokerId`: instruments allowed for that `broker_type` (same rules as `isInstrumentKindAllowedForBrokerType`); `custom` / `cash_account` must match `instruments.broker_id`; 400/404 for invalid or missing broker. Query `portfolioId`: `netQuantity` only that portfolio; else all user portfolios.
- `POST /transactions` — body includes `portfolio_id` (JSON `portfolioId`) for `USER_ID`; validates `broker_id` + `instrument_id` (kind allowed for broker type; `custom` / `cash_account` must match `instrument.broker_id`). `PATCH /transactions/:id` — same shape as `POST`; empty optional `unitPriceEur` clears column. `DELETE /transactions/:id` — 204.
- `GET /instruments/:id` — same shape as list element; 404 if missing.
- `PATCH /instruments/:id` — `cash_account`: display name, broker, `cash_currency`, `cash_geo_key`. `etf`/`stock`: `holdings_distribution_url` and/or `provider_breakdown_data_url` (`null` clears each). When either normalized URL changes for `etf`/`stock`, refresh runs like `POST /instruments/:id/refresh-distribution` (upstream failure 502/503; `manual` cache skipped); 400 for other kinds. Duplicate cash display name (case-insensitive) → 409.
- `POST /instruments` (`etf`/`stock`): `instruments.isin` from Yahoo `quoteSummary` when `extractIsinFromQuoteSummaryRaw` in `api/src/distributions/yahoo.ts` finds a valid 12-char ISIN (`normalizeIsinForStorage` in `@investments/db`); Degiro import passes `isinOverride` into `insertEtfStockFromYahoo`. ISIN may stay null if Yahoo omits it. On `POST /instruments/:id/refresh-distribution` and startup stale refresh, `isin` is backfilled when still null and extraction succeeds (does not overwrite stored ISIN).

## Imports

- **Degiro CSV:** `POST /import/degiro`, multipart `file` (UTF-8 CSV). Parser `api/src/import/degiroTransactions.ts` (`csv-parse`, `relax_column_count`). Rows normalized to 18 columns (Degiro sometimes omits empty field before Order ID UUID). Non-trade lines (missing ISIN, Reference exchange/Venue, zero qty) skipped. Same Order ID (partial fills) merged: total signed qty, VWAP; ISIN, date/time, product, exchange, venue must match or error. Upsert on `(broker_id, external_source, external_id)` with `external_source = 'degiro_csv'`; `external_id` = lowercase Order ID UUID, or sha256 of canonical row if missing. EUR only; each ISIN must resolve to one `etf`, `stock`, or `custom`. Resolution: `instruments.isin` first; else OpenFIGI (`api/src/import/openFigi.ts`) → Yahoo symbol candidates vs `instruments.yahoo_symbol`. If unmatched: 200 `{ ok: false, needsInstruments: true, proposals }` from `api/src/import/degiroInstrumentProposals.ts` (OpenFIGI + Yahoo `quoteSummary`, Yahoo search fallback). Re-`POST` with optional `createInstruments` (JSON `[{ isin, yahooSymbol, kind }]`) → `insertEtfStockFromYahoo` (`api/src/lib/createYahooInstrument.ts`). Optional `OPENFIGI_API_KEY` in root `.env` (`X-OPENFIGI-APIKEY`). Seligson-only without Yahoo ticker still need `isin` set (or no match). Success: `{ ok: true, processed, changed, unchanged }` (`changed` = rows inserted/updated with differing `RETURNING` data). Web: `/import`.
- **IBKR CSV:** `POST /import/ibkr`, multipart `file`. Parser `api/src/import/ibkrTransactions.ts`: flat **Activity** (`ClientAccountID`, `DateTime`, `ISIN`, `ExchTrade`, `TradePrice`, `CurrencyPrimary`, …) or flat **trades** (`Date/Time`, `Buy/Sell`, `Price`, `CurrencyPrimary` — no `TransactionType`). Statement-style Transaction History CSVs rejected (400). Forex skipped: `IDEALFX`, symbols like `EUR.USD`. Flat `ExchTrade`: signed `Quantity` (negative = sell); flat `Buy/Sell`: `BUY`/`SELL` + unsigned qty. `external_id`: sha256 of `isin|symbol|ExchTrade|qty|price|DateTime` (same for `Date/Time` rows). Resolution `api/src/import/ibkrResolveInstruments.ts`: unique `instruments.isin`, else `yahoo_symbol` (exact, space vs `-`/`.`, listing suffix match e.g. IBKR `SPYI` → stored `SPYI.DE`). No OpenFIGI. `unit_price_eur`: same USD→EUR stub as valuation when not EUR. Broker name `IBKR`. 400: `missingSymbols` / `ambiguousSymbols` / optional `ambiguousIsins` / `missingIsins`. Success shape matches Degiro. Web: `/import`.
- **Seligson TSV:** `POST /import/seligson`, multipart `file` (UTF-8 TSV). Parser `api/src/import/seligsonTransactions.ts`: 8 columns (portfolio export header `Salkku` … `Summa €`); empty `Tyyppi` inferred from `Summa €` sign via Merkintä/Lunastus. Upsert `(broker_id, external_source, external_id)` with `external_source = 'seligson_tsv'`; `external_id` = join of ISO date, type label, `Rahasto`, `Osuuksien määrä`. Broker row named `Seligson`. `Rahasto` must match exactly one `seligson_funds.name` for a `custom` instrument with that broker (trailing `(A)`/`(B)` ignored); else 400 `missingFundNames` / `ambiguousFundNames`. Optional `skipMissingInstruments` (`true`/`1`): import only resolvable rows; 400 if all would skip. Success may include `skippedRows`. No auto instrument creation. Web: `/import`.

## Other routes

- `GET /health` — `{ ok: true }`
- `GET /portfolios`, `POST /portfolios`, `PATCH /portfolios/:id` — `USER_ID`; `PATCH` name and/or `emergencyFundEur`; duplicate name 409 on create/patch
- `GET /brokers`, `POST /brokers`, `PATCH /brokers/:id`, `DELETE /brokers/:id` — CRUD; `DELETE` 409 if transactions or instruments remain
- `GET /transactions?portfolioId=` — `portfolioId` required; 400/404 if missing, invalid, or not owned
- `GET /positions?portfolioId=` — same; open positions + joined `instrument` per line
- `GET /instruments/lookup-yahoo?symbol=` — Yahoo metadata for new-instrument UI
- `POST /instruments/:id/refresh-distribution` — refetch distribution cache (502 upstream; 404 missing; 200 `{ ok: true, instrument }` same as `GET /instruments/:id`, or `{ skipped: true, reason }` for cash/manual)
- `DELETE /instruments/:id` — deletes transactions in one DB transaction; `distributions`, `prices`, `yahoo_finance_cache`, `seligson_distribution_cache`, `provider_holdings_cache` CASCADE from `instruments` (204). `seligson_funds` rows not deleted.

## Portfolio distributions

- `GET /portfolio/distributions` — requires `portfolioId` (same user). Returns value-weighted merged `countries` (raw keys, before geo bucketing), aggregated `regions` (geo buckets), `sectors` (non-cash only — `sectors.cash` omitted, renormalized to 100%), `totalValueEur`, `mixedCurrencyWarning` (reserved; currently always false), `assetAllocation` (equities/bonds net of embedded ETF cash `cashInFundsEur` from `sectors.cash` × position value; `cashExcessEur` / emergency split from `cash_account` vs `portfolios.emergency_fund_eur`; Yahoo/Seligson heuristics for bonds), `bucketTopHoldings` `{ regions, sectors, countries }` — bucket key → up to five `{ instrumentId, displayName, pctOfBucket }` (0–1), `positions` rows: `quantity`, `unitPriceEur`, `weight` (share of `totalValueEur`; web renormalizes within `assetClass` sections), `valueEur`, `valuationSource`, `assetClass` (`equity` \| `bond` \| `cash_account`), `instrumentId`, `displayName` — see `api/src/lib/portfolio.ts`. Holdings without distribution cache → synthetic `__portfolio_unknown__` on `countries`/`sectors` and geo bucket `unknown` on `regions` (value-weighted among non-cash only).
- Portfolio weighting and EUR valuation: `api` `lib` — read before changing FX or valuation.
- Web routes: discover from `web` source.
